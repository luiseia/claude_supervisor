## 三、Center/Around 采样策略 (P1 — 与 Plan D 并行引入)

在 `generate_occ_flow_labels.py` 的 `_compute_fine_cell_ids` 函数（约 line 245-294）中，修改返回值以标记 center cell：

**修改方案：**

1. 在返回的 `cell_ids` 基础上，额外返回 `center_cell_id`：
```python
# 在 _compute_fine_cell_ids 末尾
center_c = int(cx_img / fine_cell_w)
center_r = int(cy_img / fine_cell_h)
center_cell_id = center_r * GRID_FINE_W + center_c
return cell_ids, (cx_img, cy_img), (fine_cell_w, fine_cell_h), center_cell_id
```

2. 在 `__call__` 方法中，将 `center_cell_id` 存入 info dict：
```python
info['center_cell_id'] = center_cell_id
```

3. 在 `git_occ_head.py` 的 `_get_targets_single_based_on_bev` 中，利用 center 信息调制权重：
```python
# 在权重分配阶段 (line ~494-505)
center_cell = info.get('center_cell_id', -1)
is_center = (cid == center_cell)
center_boost = 2.0 if is_center else 0.5  # center 给 2x 权重，around 给 0.5x
final_w = base_w * center_boost
```

这样 center cell 获得更高权重（位置最可靠），around cells 降低权重（标签可能有噪声）。
